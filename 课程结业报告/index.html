1、导航条设计

一个网站的导航条可以让访问者对你的网站有一个大概的了解，同时用来对网页内容进行导航，在点击相关内容时进行跳转，这样更为方便。
运用MDB中的导航条模板，放在< header >标签中，例如下面这个：

<!--Main Navigation-->
<header>
  <!--Navbar-->
  <nav class="navbar navbar-expand-lg navbar-dark primary-color">
    <!-- Navbar brand -->
    <a class="navbar-brand" href="#">Navbar</a>
    <!-- Collapse button -->
    <button class="navbar-toggler" type="button" data-toggle="collapse" data-target="#basicExampleNav"
      aria-controls="basicExampleNav" aria-expanded="false" aria-label="Toggle navigation">
      <span class="navbar-toggler-icon"></span>
    </button>
    <!-- Collapsible content -->
    <div class="collapse navbar-collapse" id="basicExampleNav">
      <!-- Links -->
      <ul class="navbar-nav mr-auto">
        <li class="nav-item active">
          <a class="nav-link" href="#">Home
            <span class="sr-only">(current)</span>
          </a>
        </li>
        <li class="nav-item">
          <a class="nav-link" href="#">Features</a>
        </li>
        <li class="nav-item">
          <a class="nav-link" href="#">Pricing</a>
        </li>
        <!-- Dropdown -->
        <li class="nav-item dropdown">
          <a class="nav-link dropdown-toggle" id="navbarDropdownMenuLink" data-toggle="dropdown" aria-haspopup="true"
            aria-expanded="false">Dropdown</a>
          <div class="dropdown-menu dropdown-primary" aria-labelledby="navbarDropdownMenuLink">
            <a class="dropdown-item" href="#">Action</a>
            <a class="dropdown-item" href="#">Another action</a>
            <a class="dropdown-item" href="#">Something else here</a>
          </div>
        </li>
      </ul>
      <!-- Links -->
      <form class="form-inline">
        <div class="md-form my-0">
          <input class="form-control mr-sm-2" type="text" placeholder="Search" aria-label="Search">
        </div>
      </form>
    </div>
    <!-- Collapsible content -->
  </nav>
  <!--/.Navbar-->
</header>

我们可以看到导航条最重要的就是< nav >标签，其内部包含了< a >, < button >，< div > 三个标签。< a > 超链接用来实现网页的快速跳转，< button >按钮用来实现点击，< div > 用于放置各个导航超链接。

2、全屏图片背景

首先添加如下代码在< header >标签中，放在 < nav >标签之后：

<!--Mask-->
<div id="intro" class="view">
  <div class="mask">
</div>
<!--/.Mask-->

其中 view 是我们背景图片的封装器，同时可以对图片进行明暗等效果的遮罩；mask 是一个有绝对定位的元素, 它将以某种效果覆盖图片（如添加文字）；id=“intro” 这是view元素的ID。
然后在css/style.css文件中添加图片的CSS代码，如下：

html,
body,
header,
#intro {
    height: 100%;
}

#intro {
    background: url("https://mdbootstrap.com/img/Photos/Horizontal/Nature/full%20page/img%20%283%29.jpg")no-repeat center center fixed;
    -webkit-background-size: cover; /* 针对Chrome, Safari, Edge等浏览器 */
    -moz-background-size: cover;    /* 针对Firefox浏览器 */
    -o-background-size: cover;      /* 针对Operallq */
    background-size: cover;         /* 通用 */
}

此时通过ID(intro)为该元素设置背景图像的 URL；用cover来设置背景的尺寸, 它将覆盖屏幕所有可用的空间； -webkit-， -moz- ，-o-前缀是为了确保在所有浏览器都正常工作。
这样我们就能得到一张全屏背景图片，如下：

3、图片遮罩效果

在设计中，为了突出我们的主体内容，就要让图片成为我们的背景，即弱化图片效果，此时就需要对图片添加遮罩效果。添加一个简单的强暗黑遮罩效果用如下代码：

<div class="mask rgba-black-strong"

更多遮罩效果见链接：Masks。添加遮罩后我们就可以在图片上添加一些文字或图标，如下：

<div class="mask rgba-black-strong">

  <!-- Heading -->
  <h2 class="display-4 font-weight-bold white-text pt-5 mb-2">Travel</h2>

  <!-- Divider -->
  <hr class="hr-light">

  <!-- Description -->
  <h4 class="white-text my-4">Lorem ipsum dolor sit amet, consectetur adipisicing elit. Deleniti consequuntur.</h4>
  <button type="button" class="btn btn-outline-white">Read more<i class="fas fa-book ml-2"></i></button>

</div>

其中display-4是 一个好看的大标题，font-weight-bold对文本进行加粗，hr-light是一条白色分割线，btn-outline-white是一个白色的透明轮廓按钮，fa-book是 一个书本样的图标。

遇到的问题

1、图片大小问题

在有多个图片放置在一个界面时，会出现图片大小不一致的问题，这将直接影响网页的美观性。
解决办法是：通过直接设置图片的宽度 width 和高度 height 来改变图片大小，例如插入一张图片：

<img src="https://mdbootstrap.com/img/logo/mdb192x192.jpg" alt="MDB Logo" width="200" height="200">

还有一种办法是在画图软件中直接修改图片的像素：

但两种办法都会导致图片变形，所以我都尽量选择图片大小接近的图片。

2、如何通过github托管静态网页

在我们做好自己的网页之后，我们需要把它部署到一些托管服务器如GitHub上，这样别人才能通过URL访问我们的网站。我将网站部署到github上的过程参考了视频：【WEB前端设计】 第2期 通过github托管静态网页，其大致过程如下：
（1）直接上传本地文件到GitHub
首先需要创建一个GitHub账号（如果有账号就直接登录）进入GitHub的官网，点击右上角sign up进行注册

输入相关信息并进行验证，创建一个属于自己的GitHub账号

在用邮箱进行验证之后，开始创建一个新的代码仓库，注意仓库命名规则：Owner+.github.io

创建好之后，我们就可以点击Uplate files上传本地文件夹内的文件


此时我们就可以通过域名即仓库名（Owner+.github.io）访问上传的index.html文件。

（2）通过GitHub桌面端管理代码文件
我们可以通过下载GitHub桌面端即GitHub Desktop软件，对多个网页代码文件进行管理，同时能够进行实时的更新，更为方便快捷。
下载安装好GitHub Desktop后，在软件内登陆GitHub账号

登录后将服务器上的代码克隆到本地文件夹

此时我们就可以在本地文件夹对代码文件进行管理或修改，GitHub Desktop识别到变化后会给我们相应的提示，此时选择push origin上传，就可以将更新的内容上传到GitHub服务器上。

3、网页跳转的问题

网页中放置超链接的地方都可以实现网页的跳转。但在我制作的网页中，无论是导航条中还是按钮放置的超链接，访问我自己已经托管在GitHub上的网页时总会出现问题，时常不能访问。
我从网站首页跳转到子页面是能轻松做到的，但从子页面返回首页时却只是有时候可以访问，大部分时间都不会进行跳转，这个问题我尚未解决，不知道是否是GitHub网站的原因，有时GitHub的官网都无法进行访问。

个人总结

通过本次Web页面的制作实践，我了解并掌握了HTML文本和CSS标签一些相关的基本知识，了解了它们基本的页面结构。同时我能够运用VS Code软件进行一些简单的网页制作，并通过查阅资料，学会如何将静态网页部署在GitHub上，让别人能够对我制作的页面进行访问，同时我也可以在本地文件夹对我的代码文件进行管理，对网页内容进行修改、增删等操作。
此外，我发现MDB是一个非常强大的网站，它提供了非常多简洁、直观又漂亮的前端CSS开发框架，而且它还附有Web开发相关的官方教程，非常直接明了、易于理解，这让我们在进行Web开发时变得更为迅速、明朗。
